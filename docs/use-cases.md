# Use Cases

This guide maps common TB2 jobs to the right preset, transport, and review posture.

## 1. Normal Host + Guest coding loop

Use:

- preset: `Quick Pairing`
- transport: `SSE`
- forwarding: `auto_forward=true`
- review: `intervention=false`

Best for:

- delegated debugging
- code review loops
- multi-step refactors where the Host still owns final judgment

## 2. Sensitive forwarding with human approval

Use:

- preset: `Approval Gate`
- transport: `SSE`
- forwarding: `auto_forward=true`
- review: `intervention=true`

Best for:

- shell commands that may mutate the repo
- dependency changes
- migrations or CI repair

## 3. MCP-first orchestration from another AI client

Use:

- preset: `MCP Operator`
- browser UI mainly for status, room, and pending queue visibility
- MCP tools for launch and control

Best for:

- Codex as planner, Guest in panes
- Claude Code or Gemini driving tool calls
- reproducible local automation with a stable control plane

## 4. Terminal incident recovery

Use:

- preset: `Diagnostics`
- transport: `room_poll` or `SSE`
- focus on capture, interrupt, and status

Best for:

- a Guest tool hanging
- confusing pane output
- validating shell startup and backend behavior

## 5. CLI-only operator workflow

Use:

- `python -m tb2 room watch --room-id <ROOM_ID>`
- `python -m tb2 capture --target <PANE>`
- `python -m tb2 send --target <PANE> --text "..."`

Best for:

- SSH-only environments
- low-friction demos
- recovery on machines without a browser

## 6. Dense handoff review loop

Use:

- preset: `Handoff Radar`
- transport: `SSE`
- forwarding: `auto_forward=true`
- review: `intervention=true`

Best for:

- repeated review / approve / reject loops
- workflows where the Host watches both the room and pending queue together
- shell-heavy Guest sessions where handoffs arrive frequently

## 7. Low-noise pairing and operator coaching

Use:

- preset: `Quiet Loop`
- transport: `SSE`
- forwarding: `auto_forward=true`
- review: `intervention=false`

Best for:

- pairing sessions where the operator mainly sends short guidance
- demos where diagnostics should stay out of the way
- situations where launch plus live room are the only primary controls

## 8. Host-led coordination shift

Use:

- preset: `Mission Control`
- transport: `WebSocket`
- forwarding: `auto_forward=false`
- review: `intervention=false` by default

Best for:

- a Host AI supervising topology, diagnostics, and room traffic at the same time
- investigations where raw status is as important as the live room
- long-running sessions that need a broader control-plane view

## 9. When to switch from simple to advanced mode

Stay in the default task preset when:

- pane IDs were generated by the same launch flow
- you only have one active bridge
- you do not need custom transport tuning

Open advanced controls when:

- you need explicit `backend_id`, `bridge_id`, or `room_id`
- you want to pin `transport`
- you are mixing browser UI with MCP automation
- you are debugging pane naming or delivery routing
